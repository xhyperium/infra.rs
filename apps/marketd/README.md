# marketd

市场数据服务的二进制组合根，负责装配 `market_data` 的源、检查点与事件下游，并处理进程关闭。

## 边界

- 负责进程生命周期和依赖装配，不承载市场数据领域规则。
- 当前运行路径使用仓库内 fixture；真实 provider 接入不属于本 crate 的现有职责。
- 内部模块不提供稳定的库 API；可执行入口为 `cargo run -p xhyper-marketd`。


## 非职责

- 不承载市场数据领域规则或协议解析实现。
- 不提供稳定的库 API（内部模块面向二进制组装）。
- 当前路径不承诺生产级真实 provider 完整性（见限制）。

## 限制与安全

- 二进制仅负责生命周期与装配；配置错误应 fail-fast。
- 真实 provider / 凭据不得写入仓库；需外部配置注入。
- 优雅关闭须避免泄漏任务（孤儿 task）。

## 依赖方向

仅向下依赖 `market_data` 服务能力和运行时依赖。设计说明见 [`docs/README.md`](docs/README.md)。
