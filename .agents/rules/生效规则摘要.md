> **定位**：两屏内可读完的本仓硬规则 + 审查档位 + 合入门禁。  
> **上位**：宪章 `docs/constitution/` · 组织 `~/.claude/rules/` · 细则见本目录其它文档。  
> **冲突裁决**：组织 P0 不可削弱；细则不得与本摘要红线矛盾。

## 2. 合入前必跑（唯一清单）

### 2.1 本地最小集（声明完成前）

按变更类型选一列，**不要**混用多套清单：

| 变更类型 | 必跑 |
|----------|------|
| **仅文档 / 规则 / SSOT 文案** | 改动文件 `markdownlint` + `cspell`（或依赖 CI 校验 job）；触及 crates 看板时 `node scripts/docs/gen-crate-status.mjs` |
| **Rust 代码 / Cargo** | 下方「Rust 四件套」 |
| **脚本 / hooks** | `node scripts/self-test.mjs --lint-only`（改逻辑再跑对应 `*.test.mjs`） |
| **不确定 / 跨面** | `make ci` + `make check` |

**Rust 四件套**（与宪章 §5 / `make ci` 对齐）：

```bash
cargo fmt --all -- --check
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test --workspace --all-features
cargo deny check
```

触及依赖声明时追加：

```bash
node scripts/quality-gates/check-workspace-deps.mjs
```

crate 交付性更新时：对应 package **PATCH +1**（`VERSIONING.md`）。

可选一键：`make ci`（fmt + lint + test + deny）、`make check`（宪章脚本）。

### 2.2 分支保护 required checks（合入硬门槛）

Ruleset 当前 **required**（名称须完全一致）：

1. `Constitution Check`
2. `Template Validation`

其余 workflow（org-rust fmt/clippy/test、Markdown、拼写、Harness 等）多为 **信号 / 推荐**；应尽量绿，但**以仓库 Ruleset 配置为准**。  
权威快照：`docs/status/CONFIG_SUMMARY.md`（过期时以 GitHub Ruleset 为准）。

### 2.3 合入路径（标准）

```text
worktree 开发 → 本地 §2.1 → push → 开 PR
  → required checks 绿
  → 至少 1 次非作者 approve（CODEOWNERS / 第二维护者）
  → squash merge（或 --auto）
  → 清理 worktree / 分支
```

## 3. 审查档位（禁止默认 10 轮）

| 档位 | 适用 | 审查深度 | 默认轮次 |
|------|------|----------|----------|
| **T0 trivial** | 单文件 typo/注释/索引；≤1 文件且净变更很小 | diff 通读 + 链接/格式 | **1** |
| **T1 standard** | 常规功能/修复/文档规则；≤ 数个 crate | Standards + Spec 各一过；门禁绿 | **1–2** |
| **T2 production-ready** | L1–L5 签核、package 晋级、交易/资金相关、跨层 wire | 分维度审查 + 证据包 + Maintainer 签核 | **按签核清单**，非固定 10 轮 |
| **T3 incident / security** | 安全事故、密钥、供应链 CRITICAL | 安全审查 + 人工 Maintainer | 按事件，须收敛 |

**禁止**：

- 无 T2/T3 理由默认「十轮 review」
- 用轮次数字代替证据（命令输出 / CI URL / 对齐文）
- 审查者修改被审源码并自批（宪章 C-3）

落盘：常规结论可在 PR 评论；T2+ 证据放 `docs/report/{YYYY-MM-DD}/` 或 `docs/plans/releases/`。

## 4. pr-auto-approve 约束

技能：`.claude/skills/pr-auto-approve/`（第二身份 `@liukongqiang5`，**不是**作者自批）。

| 允许 | 禁止 |
|------|------|
| **用户明确要求** approve / auto-approve / 合入 | Agent **默认**、静默、批量批准无关 PR |
| required checks 已绿或用户接受 `--auto` 等待 | 检查仍红时「假装可合」 |
| 作者 ≠ `liukongqiang5` | 用该 token 批 **自己** 的 PR |
| 合入后在回报中写明身份、PR、SHA | 打印 token / 写入仓库 |

**默认策略**：未获用户明确合入/批准指令时，Agent **只**开 PR、贴检查状态，**不**调用 auto-approve。

审计最低信息（对话回报即可）：`PR#` · head SHA · 批准身份 · 用户是否明示。
