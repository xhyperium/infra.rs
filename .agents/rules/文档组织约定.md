# 文档组织约定

本文档定义 infra.rs 仓库的文档目录结构与命名规范。

---

## §1 报告文档保存路径

### 规则

| 项目 | 约定 |
|------|------|
| 根路径 | `docs/report/{YYYY-MM-DD}/` |
| 日期格式 | ISO 8601 日期（四位年-两位月-两位日），如 `2026-07-23` |
| 必含文件 | 每个日期目录下必须包含 `README.md` 索引文件 |
| 子目录 | 允许按需创建多级子目录（如 `round-01/`、`evidence/`） |

### 示例

```text
docs/report/
├── 2026-07-21/
│   ├── README.md
│   └── round-01/
│       └── ...
├── 2026-07-22/
│   ├── README.md
│   ├── round-01/
│   └── round-02/
└── 2026-07-23/
    ├── README.md
    └── ...
```

### 已废弃格式

- `YYYY-MM-DD-slug`（如 `2026-07-22-defer-close/`）— **禁止**新建，已有目录视情况迁移或保留原文不动

---

## §2 规则文档路径

| 项目 | 约定 |
|------|------|
| 项目规则 SSOT | `.agents/rules/` |
| 兼容旧路径 | `docs/governance/`（仅 stub 重定向） |
| 组织全局规则 | `~/.claude/rules/`（`xhyperium/.github` `rulesets/`，不入库副本） |
| 工程宪章 | `docs/constitution/` |

新增或修订项目规则时，**只**改 `.agents/rules/`，并同步更新该目录 `README.md` 索引。
