# eastmoney — 分阶段计划

本计划不批准任何仓库实现路径。所有阶段均先产出文档、脱敏 fixture 和离线测试；provider crate 只有在 manifest 写入获批路径后才能创建。

| 阶段 | 交付物 | 退出条件 |
|---|---|---|
| P0 合同核验 | 官方来源、授权边界、字段/单位清单 | 人工审查通过，证据含日期和 SHA-256 |
| P1 离线模型 | `domain_macro` 映射、严格解析、错误码 | fixture、坏输入、缺失值和重复身份测试通过 |
| P2 适配器评审 | 批量与分页的纯策略 | 不含访问方式规避；路径已获批并登记 |
| P3 运行时实现 | 独立 provider crate | Cargo member、日志脱敏、质量门禁和合规证据齐全 |

达到配额、拒绝或挑战时停止并返回稳定错误；不得通过改变客户端行为继续访问。任何具体代码路径在批准前都使用“未批准实现路径”文字，不作为 evidence。
