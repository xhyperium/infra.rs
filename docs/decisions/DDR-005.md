# DDR-005: 统一 target-dir 到 .cargo/target/

**日期**: 2026-07-21
**状态**: 已接受

## 上下文

Rust 默认将构建产物放 `target/`。在 workspace 根目录运行 `cargo build` 时产物在 `./target/`，但在子目录运行可能产生额外 `target/` 目录。CI 缓存策略和本地清理需一致性。

## 选项

| 选项 | 优点 | 缺点 |
| ------ | ------ | ------ |
| 默认 `target/` | 零配置 | 多位置 target 导致缓存碎片 |
| 环境变量 `CARGO_TARGET_DIR` | 灵活，但需每个环境设置 | 容易遗忘配置 |
| **.cargo/config.toml 统一设置** | 仓库内声明，CI 和本地一致 | 对习惯默认 target/ 的开发者有学习成本 |

## 决策

通过 `.cargo/config.toml` 设置 `target-dir = ".cargo/target/"`，在仓库内统一所有构建产物路径。

## 后果

- **正面**: CI 和本地开发使用相同路径，Swatinem/rust-cache 命中率更高
- **正面**: `.cargo/` 整体 gitignored，清理只删一个目录
- **正面**: 工具缓存（advisory-db、mutants）也可放在 `.cargo/cache/` 下
- **负面**: 开发者需习惯构建产物不在根目录 `target/`
