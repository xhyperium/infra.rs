# DDR-003: serde 手动实现 Error 序列化

**日期**: 2026-07-21
**状态**: 已接受

## 上下文

`infra_core::Error` 需要在网络边界（HTTP API、IPC、日志导出）传输。`serde::Serialize`/`Deserialize` 是 Rust 生态标准。对于 `std::io::Error`，简单的 derive 实现会丢失 `source()` 链。

## 选项

| 选项 | 优点 | 缺点 |
| ------ | ------ | ------ |
| 仅 derive Serialize/Deserialize | 简单，零手写代码 | IO error 链丢失，`error_kind` 无法保留 |
| serde 的 remote derive | 保持外部类型序列化 | 不适用于包含动态 source() 链的 io::Error |
| **手动实现** | 完整保留 error_kind 和 source() 链 | 代码量较大（约 150 行） |

## 决策

为 `Error` 枚举**手动实现 `serde::Serialize` 和 `serde::Deserialize`**，通过 `ChainNode` 辅助结构体完整保留 IO 错误链。

## 序列化格式

```json
// Io 变体
{"kind": "Io", "error_kind": "NotFound", "chain": ["top level", "source 1", "root"]}

// 其他变体
{"kind": "Config", "message": "missing field host"}
```

## 后果

- **正面**: 完整的跨边界错误传播能力
- **正面**: 反序列化后可重建完��的 `source()` 调用链
- **负面**: 约 150 行手动实现代码需维��
- **负面**: 新增 `io::ErrorKind` 变体时需更新 match 臂
