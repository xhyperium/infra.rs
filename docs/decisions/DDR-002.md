# DDR-002: thiserror v2 作为错误派生方案

**日期**: 2026-07-21
**状态**: 已接受
**修订**: 2026-07-21 — 适用范围从已移除的 `infra-core` 调整为 workspace 库 crate

## 上下文

workspace 库 crate 需要一个类型安全的错误处理方案。Rust 生态中主要有三种方式：`std::error::Error` 手动实现、`thiserror` 派生宏、`anyhow` 动态错误。

## 选项

| 选项 | 优点 | 缺点 |
| ------ | ------ | ------ |
| 手动实现 `std::error::Error` | 无外部依赖 | 样板代码多，易出错 |
| **thiserror v2** | 派生宏减少样板；类型安全；社区标准 | 增加编译时间，额外依赖 |
| anyhow | 极简 API，适合应用层 | 类型擦除丢失错误信息；不适合库 |

## 决策

采用 **thiserror v2** 作为 workspace **库 crate** 的错误派生方案（应用侧可用 `anyhow`）。

## 后果

- **正面**: `#[derive(Error)]` 一行宏替代 20+ 行手动实现
- **正面**: `#[from]` 属性自动实现 `From` 转换，支持 `?` 传播
- **正面**: `#[error("...")]` 统一 `Display` 格式
- **负面**: 增加编译依赖（thiserror + thiserror-impl + syn 等）
