# DDR-006: Beads (bd) 作为任务跟踪系统

**日期**: 2026-07-21
**状态**: 已接受

## 上下文

infra.rs 是多 AI 代理（Claude Code、Codex、Copilot）协作的元仓库。传统 GitHub Issues 无被 AI 代理以结构化方式读写。需要一跨模型、可编程的任务跟踪方案。

## 选项

| 选项 | 优点 | 缺点 |
| ------ | ------ | ------ |
| GitHub Issues | 无需额外工具；与 PR 集成 | AI 编程读写受限；缺少结构化依赖 |
| 纯 Markdown TODO | 极简 | 无依赖跟踪；多代理冲突风险 |
| **Beads (bd)** | CLI 可编程；Dolt DB 结构化存储；跨模型共享 | 额外工具学习成本；需本地 Dolt 运行环境 |

## 决策

采用 **Beads（`bd` CLI）** 作为任务跟踪系统。本地使用 Dolt 数据库，通过 `refs/dolt/data` 在 git remote 上同步。`.beads/` 目录 gitignored，不进入版本库。

## 后果

- **正面**: `bd create/update/close/dep` 命令行全流程覆盖
- **正面**: 结构化依赖跟踪（`bd dep add`），支持 blocking/blocked 关系
- **正面**: 跨模型共享（Claude Code 读写，Codex 读写，Copilot 只读）
- **负面**: 团队成员需额外安装和学习 `bd` 工具
- **负面**: Dolt 数据库管理与 git 工作流的集成需额外理解
