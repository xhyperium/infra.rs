# DDR-008: 12 钩子 + 46 技能 + 28 Codex 代理的 AI 治理分层

**日期**: 2026-07-21
**状态**: 已接受

## 上下文

infra.rs 的核心使命之一是作为 AI 编码助手的治理中��。需要定义多个代理系统��Claude Code、Codex、Copilot）的协作规则，避免冲突和配置漂移。

## 决策

采用三层 AI 治理架构：

- **Claude Code**: 主执行代理 — 12 个生命周期钩子控制会话全流程
- **Codex**: 编排派工代理 — 28 个专业代理处理不同类型任务
- **Copilot**: 补充建议代理 — 自动管理，不参与写入

所有技能定义以 `.claude/skills/` 为 SSOT。`.codex/.agents/skills/` 为自动投影，禁止手工编辑。

## 钩子生命周期分层

```text
SessionStart: session-context + bd prime（上下文注入��
PreToolUse:   pre-tool-check + edit-guard + count-guard（操作前校验）
PostToolUse:  post-tool-check + edit-guard-reset + link-check（操作后检查）
PreCompact:   pre-compact（压缩前状态保留）
Stop:         session-review + version-guard + branch-protect（会话结束审查）
```

## 后果

- **正面**: Claude Code 会话全生命周期可审计
- **正面**: SSOT 防止技能定义在多代理间分叉
- **正面**: 分层设计使每个代理的能力边界清晰
- **负面**: 配置文件数量多（86+ 个），维护成本高
- **负面**: 新成员理解全貌需要时间
