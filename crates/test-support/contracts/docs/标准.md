## 1. 需求标准

### 目标

contract-testkit 为 contracts 各 trait 提供 Fake 实现和一致性测试套件（SPEC-TESTKIT-002 §3.2）。

### 边界

- **包含**：各 trait 的 Fake 实现、一致性测试套件（per-trait conformance suite）
- **不包含**：生产实现、性能测试、网络通信
- **依赖**：contracts、kernel、async-trait、tokio 等

### 约束

- dev-dep 仅用于测试
- 所有 Fake 实现在内存中完成
- 资源型 suite 必须接收调用方唯一标识，或由合法 `FixtureNamespace` 派生；禁止隐式时间、随机数和环境变量
- 每个被选中的 trait 必须同时有 reference 路径与能被 suite 杀死的 broken implementation
- production normal/build 图（含 all-features）不得包含 test-support package

## 2. 设计标准

```text
contract-testkit

├── suite/       — 每个 trait 一个普通函数 suite

└── tests/       — reference self-test + broken implementation matrix
```
