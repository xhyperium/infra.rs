## 变更摘要

- 增加 reqwest/rustls HTTPS 与可选 PEM CA。
- 远程 HTTP、CA/明文冲突、非法环境值与零 deadline 在连接前 fail-closed。
- 请求错误保留 source，密码与完整 URL 不进入错误上下文。
- 临时 CA/服务端证书实验覆盖受信 CA 成功与错误 CA 握手失败。
- `HTTP_PORT`/`PORT` 兼容别名冲突 fail-closed；错误响应只保留状态和数字错误码，
  不回显 SQL、payload 或认证正文。

## 验证

```bash
cargo test -p clickhousex --all-targets
cargo clippy -p clickhousex --all-targets -- -D warnings
node scripts/clickhouse-https-conformance.mjs
```

## 未承诺

该实验只证明客户端 TLS/CA；真实 ClickHouse 集群 TLS、mTLS、复制、HA 与 native 9000 未承诺。
