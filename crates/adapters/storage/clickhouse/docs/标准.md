# clickhousex 标准文档

**包名**：`clickhousex` v0.3.2｜**角色**：ClickHouse HTTP(S) 分析存储适配器｜**状态**：声明范围内可受控使用，package stable 未宣称

## 能力与边界

- 默认面为 reqwest `ClickHousePool` / `ClickHouseClient`，支持 ping、文本/行查询、JSONEachRow 与有界分块写入。
- `AnalyticsSink` 写入调用方管理的表；内存实现仅在 `scaffold` feature 下可用。
- Semaphore 约束并发请求，连接、请求和池获取均有非零截止时间；close 后拒绝新请求。
- 不提供原生 9000 协议、DDL/migration 所有权、Cluster/HA 或 exactly-once。

## 安全

- loopback 可显式 HTTP；远程 HTTP 在配置阶段 fail-closed。
- HTTPS 使用 rustls roots，可追加 PEM CA；空/非法/错误 CA 和主机名错误均 fail-closed。
- 密码和完整 URL 不进入 Debug 或错误上下文；结构化 SQL 标识符必须通过现有校验。
- `HTTP_PORT` 为主变量，兼容 `PORT`；双设不同值时 fail-closed。
- 错误响应只读取 4096 字节前缀用于数字错误码分类，对外不回显 SQL/payload/认证正文。

## 验收

```bash
cargo test -p clickhousex --all-targets
cargo clippy -p clickhousex --all-targets -- -D warnings
node scripts/clickhouse-https-conformance.mjs
```

本地 TLS HTTP 协议实验只证明 HTTPS/CA 客户端合同，不证明真实 ClickHouse 集群、复制、
故障切换或 mTLS；真实 ClickHouse TLS live 仍为 OPEN。
