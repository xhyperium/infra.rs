# taosx 标准文档

**包名**：`taosx` v0.3.2｜**角色**：TDengine REST 时序适配器｜**状态**：声明范围内可受控使用，package stable 未宣称

## 能力与边界

- SQL 统一走 REST；`NativeWs` 只做受 deadline 约束的 WS 可达性探测，不是 Native SQL。
- `Tick.ts` 输入为纳秒，连接时探测并校验数据库精度；探测失败或显式配置不一致均 fail-closed。
- bid/ask 使用 `NCHAR(64+)` Decimal 文本，存量 `DOUBLE` schema 拒绝写入。
- response bytes、query rows、batch rows/bytes、in-flight 与 close drain 具有硬上限。
- 远程端点强制 TLS 与非空认证；严格 loopback 才允许明文开发连接。

## 可靠性与安全

- 所有标识符先校验；symbol 到子表名使用无碰撞编码。
- REST 禁止 redirect；密码不进入 Debug 或错误上下文。
- 当前批写不自动重试，避免结果不明时重复写；调用方必须按业务幂等策略处理。

## 验收

```text
cargo test -p taosx --all-targets
cargo clippy -p taosx --all-targets -- -D warnings
node scripts/taos-live-conformance.mjs
```

固定 digest 单节点 live 只证明 REST/NCHAR Decimal 往返；Native SQL、WS 长会话认证、HA、
自动幂等重试与 package stable 保持 NO-GO。
