# redisx 标准文档

**包名**：`redisx` v0.3.2 | **角色**：Redis 存储适配器 | **层级**：Production-default

---

## 1. 需求标准

### 目标

redisx 提供生产级 Redis 客户端，实现 KeyValueStore 合约 + 可选 PubSub，支持 Cluster/Sentinel/TLS + resiliencx 集成。

### 边界

- **包含**：RedisPool（连接池）、RedisKeyValueStore（KV 实现）、PubSub（feature-gated）、Cluster/Sentinel/TLS 配置
- **不包含**：原生 Redis 命令透传、��叉槽扫描、完整管理命令
- **依赖**：redis crate、resiliencx(tokio)、contracts

### 核心接口

| 符号 | 说明 |
|------|------|
| RedisPool | 连接池管理层 |
| RedisKeyValueStore | KeyValueStore trait 实现 |
| redisx::Config | 连接配置（cluster/sentinel/single） |

---

## 2. 设计标准

### 架构

```text
redisx
├── pool        — RedisPool（连接池 + resiliencx 重试）
├── kv          — RedisKeyValueStore implements KeyValueStore
├── pubsub      — PubSub 实现（feature "pubsub"）
└── config      — 连接配置（cluster/sentinel/TLS）
```text

### 数据流

```text
调用方 → RedisKeyValueStore.get(key) → RedisPool.get_conn → Redis GET key → 响应解码
调用方 → RedisKeyValueStore.set(key, value) → RedisPool.get_conn → Redis SET key value
```text

## 5. 性能标准

| 操作 | 延迟目标 |
|------|----------|
| 单次 GET | < 5ms（网络RTT内） |
| 单次 SET | < 5ms |
| 连接池获取 | < 100μs |

## 6. 安全标准

### 密钥管理 — Redis 密码通过配置注入、TLS 证书管理、禁止日志回显密码

### 网络安全 — TLS 支持（feature "tls"）、Cluster/Sentinel 认证

## 7. 可靠性标准

### 连接池 — 最大连接数/最小空闲连接/连接超时、连接健康检查、异常连接自动驱逐

### 重试 — 通过 resiliencx 集成、Transient 错误重试（连接超时/拒绝）

### 故障转移 — Cluster 自动发现节点/Sentinel 自动切换

## 8. 可维护性 — RedisPool 接口稳定、配置通过 configx 注入

## 9. 数据标准 — 所有值序列化为字节（serde 由上层负责）、无时区问题
