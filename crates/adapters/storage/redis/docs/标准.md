# redisx 标准文档

**包名**：`redisx` v0.3.4（未发布候选） | **角色**：Redis 存储适配器 |
**层级**：Production-default

---

## 1. 需求标准

### 目标

redisx 提供生产默认 Redis 客户端，实现 KeyValueStore 合约 + 可选 PubSub。Cluster/Sentinel/TLS
命令代码路径与 resiliencx 集成已存在，但真实拓扑 live 仍 OPEN。

### 边界

- **包含**：RedisPool（连接池）、RedisKeyValueStore（KV 实现）、PubSub（feature-gated）、Cluster/Sentinel/TLS 配置
- **不包含**：原生 Redis 命令透传、叉槽扫描、完整管理命令
- **依赖**：redis crate、resiliencx(tokio)、contracts

### 核心接口

| 符号 | 说明 |
|------|------|
| RedisPool | 连接池管理层 |
| RedisClient | KeyValueStore trait 实现 |
| redisx::Config | 连接配置（cluster/sentinel/single） |

---

## 2. 设计标准

### 架构

```text
redisx
├── pool        — RedisPool（连接池 + resiliencx 重试）
├── kv          — RedisKeyValueStore implements KeyValueStore
├── pubsub      — PubSub 实现（feature "pubsub"）
└── config      — 连接配置（cluster/sentinel/TLS）
```

### 数据流

```text
调用方 → RedisClient.get(key) → RedisPool 命令 lane → Redis GET key → 响应解码
调用方 → RedisClient.set(key, value) → RedisPool 命令 lane → Redis SET key value
```

## 7. 可靠性标准

### 连接池 — 最大连接数/最小空闲连接/连接超时、连接健康检查、异常连接自动驱逐

### 重试

RedisClient 的 budget 路径使用显式 `RetrySafety`：读操作为 `ReadOnly`，固定无 TTL 写为
`Idempotent`，DEL/PEXPIRE/相对 TTL SET 为 `UnsafeSideEffect`。旧无 safety wrapper 仅兼容保留。
`max_attempts == 0` 不得归一化为 1，必须在 future/driver 前返回 `Invalid`。legacy async wrapper
委托统一 unchecked core 只收敛 budget 错误与 attempt 语义，不获得 safety 保证。

固定无 TTL 写包括 SET 与 MSET。`RedisOperation::Set` 因不携带 TTL 参数而保守保持
`AmbiguousWrite`；client 按实际 TTL 参数细分。PUBLISH 不进入自动预算重试。

### 故障转移 — Cluster/Sentinel 命令连接路径已实现；真实切换与恢复证据 OPEN
