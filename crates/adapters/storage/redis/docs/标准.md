# redisx 标准文档

**包名**：`redisx` v0.3.6（未发布候选） | **角色**：Redis 存储适配器  
**层级**：Production-default（Standalone P0；非 package stable）

---

## 1. 需求标准

### 目标

redisx 提供生产默认 Redis 客户端，实现 `KeyValueStore` 合约 + 可选 `PubSub`。  
Cluster / Sentinel / TLS **命令代码路径**存在，但真实拓扑 live 仍 **OPEN**。

### 边界

- **包含**：`RedisPool`、`RedisClient`（KV）、`RedisPubSub`（feature `pubsub`）、Standalone live 入口
- **不包含**：可靠消息投递、Cluster/Sentinel 生产宣称、Streams/锁（P2 OPEN）
- **依赖**：`redis` crate、`resiliencx`（tokio）、`contracts`

### 核心接口

| 符号 | 说明 |
|------|------|
| `RedisPool` | 连接资源 + Semaphore 背压 + close |
| `RedisClient` | `KeyValueStore` + KV 扩展 |
| `RedisConfig` / `RedisConfigBuilder` | 私有字段配置（mode / TLS / 超时） |

---

## 2. 设计标准

### 架构

```text
redisx
├── pool        — RedisPool（命令 lane + 背压）
├── client      — RedisClient implements KeyValueStore
├── pubsub      — RedisPubSub（feature "pubsub"）
├── config      — RedisConfig（Standalone/Cluster/Sentinel）
├── error_map   — redis 错误 → kernel::ErrorKind
└── resilience  — 重试安全分类
```

### 数据流

```text
调用方 → RedisClient.get(key) → RedisPool 命令 lane → Redis GET → 解码
调用方 → RedisClient.set(key, value) → RedisPool 命令 lane → Redis SET
```

## 3. 测试与证据

| 类型 | 命令 | 说明 |
|------|------|------|
| 离线 | `cargo test -p redisx --all-targets` | default / `--features pubsub` |
| live | `cargo test -p redisx --test live_kv -- --ignored` | 需 `FOUNDATIONX_REDISX_*` |
| bench | `cargo bench -p redisx --bench kv_hot_path` | 有界热路径 |

当前本地离线约 49/54 passed（default / pubsub）；live 需外部 Redis，不进默认 CI 绿。

- package stable / crates.io
- Cluster · Sentinel · TLS **live 已验证**
- Pub/Sub 必达 / 断线自动恢复
