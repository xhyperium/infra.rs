# redisx 0.3.2

| 字段 | 值 |
|------|-----|
| 日期 | 2026-07-22 |
| 范围 | 内部 workspace（`publish = false`） |

## 变更摘要

- **历史事实**：0.3.2 的 `RedisClient::with_retry_budget` 使用未接收 `RetrySafety` 的
  `with_budget_async`；该路径现明确归类为 unchecked compatibility。
- round-03 候选已把真实 client 迁移到 safety-aware wrapper，并按操作语义分类；不追写成
  0.3.2 当时已经安全。
- **显式 API**：`get_with_budget` / `set_with_budget` 始终走预算环。
- **历史 async helpers**：`with_budget_async` / `with_budget_async_noop`（unchecked compatibility）。
- live I/O 仍仅 `tests/live_*`（`#[ignore]`）；默认 `--lib` 不碰外部 Redis。

## 验证

```bash
cargo test -p redisx --lib
cargo clippy -p redisx --all-targets -- -D warnings
```
