## 变更摘要

- PubSub 复用 pool 的 ACL/TLS/deadline 配置，不再从环境重建；非 Standalone 拓扑 fail-closed。
- seed URL 的 Debug、endpoint 与错误上下文统一脱敏。
- 新增显式 `RetrySafety` 的 sync/async generic budget wrapper。
- RedisClient 真实 budget 路径按操作语义迁移到安全 wrapper：
  - GET/EXISTS/PTTL/MGET 为 `ReadOnly`；
  - 无 TTL SET/MSET 为 `Idempotent`；
  - DEL/PEXPIRE/相对 TTL SET 为 `UnsafeSideEffect`。
- 旧 `with_budget*` / `with_retry*` 保留为 unchecked compatibility。
- `with_retry_budget(..., 0)` 保留 0，并在 GET/写路由 future 与 driver 前返回 `Invalid`。
- legacy async budget wrapper 委托统一 unchecked core，收敛标准 budget 错误与失败 attempt 观测；
  兼容身份不变。
- dev standalone ping/stats/close live 通过。

## 未承诺

Cluster/Sentinel/TLS live、故障切换、PubSub 断线重订阅与可靠投递保持 OPEN。
