# ossx 标准文档

**包名**：`ossx` v0.3.2 | **角色**：阿里云 OSS 对象存储适配器 | **层级**：Production-default

---

## 1. 需求标准 — ObjectStore 实现 + 分片上传 + resiliencx 重试

## 2. 设计标准 — OssObjectStore（implements ObjectStore）+ 分片上传器 + OSS V1 签名

## 3. 编码标准 — reqwest 封装、完整 rustdoc

## 4. 测试标准

- 单测（mock HTTP 响应）
- live 测试（需真实 OSS bucket，feature "live"）
- 签名生成验证（与阿里云官方签名工具对比）

## 5. 性能标准

| 操作 | 延迟目标 |
|------|----------|
| 文件上传(1MB) | < 1s |
| 文件下载(1MB) | < 500ms |
| 分片上传(100MB) | < 10s |

## 6. 安全标准（重点）

- **OSS V1 签名**：HMAC-SHA1 + 请求头规范排序 + CanonicalizedResource
- **密钥管理**：AccessKey ID + Secret 通过配置注入
- **禁止日志回显**：完整密钥禁止输出到日志
- **Bucket Policy**：最小权限原则
- **TLS**：强制 HTTPS

## 7. 可靠性标准

- resiliencx 重试集成
- 分片上传断点续传
- 超时/取消处理
- HTTP 错误分类映射

## 8. 可维护性 — ObjectStore trait 可替换

## 9. 数据标准

- 文件存储：二进制 blob
- 元数据：Content-Type、Content-MD5
- key 命名规范

## 10-11. 风控/回测 — N/A

## 12. 上线 — 上传/下载延迟监控、OSS 费用监控

## 13. 运维 — Bucket 生命周期管理、跨域配置（CORS）、版本控制

## 14. 文档 — config.md + operations.md + usage.md

## 15. 交付 — feature "scaffold" | 依赖：reqwest + hmac + sha1

## 16. 评分 — 4/5
