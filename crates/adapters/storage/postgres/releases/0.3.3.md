## 变更摘要

- 远程 PostgreSQL 仅允许 `sslmode=require`；本机/Unix 才允许 disable/prefer。
- 增加 pool acquire 与 SQL/事务终结内部 deadline，并下发服务端 `statement_timeout`。
- RAII 取消守卫覆盖内部/外层 timeout、future drop 与任务 abort；未知状态连接均从池分离。
- 事务操作进入可取消 await 前预置 `Failed`，仅在连接安全恢复后回到 `Active`。
- 服务端语句错误保持 rollback-only `Failed`；允许显式回滚但拒绝 COMMIT，避免伪报提交。
- Active transaction 的 Drop 直接关闭已分离 session，由 PostgreSQL 回滚，不启动无监督任务。
- 旧 raw client/pool 与 `database_url` 保留一个 deprecation 周期；raw client 强制脱池，
  raw pool 返回关闭的隔离池，正式路径不能绕过 deadline 与取消守卫。
- `DATABASE_URL` 仅接受 allowlist 参数，且与结构化字段不一致时 fail-closed。
- `TransactionRollbackFailure` 结构化保留业务错误与 rollback 错误两个 source 分支。
- COMMIT 超时明确返回“结果未知”，禁止伪报回滚。
- 固定摘要 PostgreSQL 17 实验覆盖池饱和、`pg_sleep` 截止、连接隔离与恢复。
- 新增显式 `RetrySafety` 的 sync/async generic budget wrapper。
- 任意 SQL 未经调用方证明时视为 `UnsafeSideEffect`；多次尝试在 operation 前拒绝。
- 旧 `with_budget*` / `with_retry*` 保留为 unchecked compatibility；legacy async 路径委托统一
  unchecked core，标准化 budget exhaustion 与失败 attempt 观测。
- `PostgresPool` 当前没有 budget 字段或自动重试接线。
