## 变更摘要

- 远程 PostgreSQL 仅允许 `sslmode=require`；本机/Unix 才允许 disable/prefer。
- 增加 pool acquire 与 SQL/事务终结内部 deadline，并下发服务端 `statement_timeout`。
- RAII 取消守卫覆盖内部/外层 timeout、future drop 与任务 abort；未知状态连接均从池分离。
- 事务操作进入可取消 await 前预置 `Failed`，仅在连接安全恢复后回到 `Active`。
- 服务端语句错误保持 rollback-only `Failed`；允许显式回滚但拒绝 COMMIT，避免伪报提交。
- Active transaction 的 Drop 直接关闭已分离 session，由 PostgreSQL 回滚，不启动无监督任务。
- 移除底层 deadpool 池访问器，禁止调用方绕过 deadline 与取消守卫。
- `DATABASE_URL` 不再作为第二份连接配置保留，避免 URL 与公开 TLS 字段不一致。
- COMMIT 超时明确返回“结果未知”，禁止伪报回滚。
- 固定摘要 PostgreSQL 17 实验覆盖池饱和、`pg_sleep` 截止、连接隔离与恢复。

## 验证

cargo test -p postgresx --all-targets
