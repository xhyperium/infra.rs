# postgresx 标准文档

**包名**：`postgresx` v0.3.2 | **角色**：PostgreSQL 存储适配器 | **层级**：Production-default

---

## 1. 需求标准 — PostgreSQL 池/SQL 适配器，支持 SSL/TLS + resiliencx 集成 + PgRepository

## 2. 设计标准 — PgPool（deadpool-postgres）+ PgRepository（implements Repository）+ Tx 管理 + SSL require

## 3. 编码标准 — 完整的 SQL 参数化查询、XError 映射、禁止 SQL 拼接

## 4. 测试标准 — 单测（mock）+ live 测试 + SQL 注入测试

## 5. 性能标准

| 操作 | 延迟目标 |
|------|----------|
| 连接获取 | < 5ms |
| 简单查询 | < 20ms |
| 批量写入 | < 50ms/批 |

## 6. 安全标准

- **注入防护**：所有查询使用参数化绑定（$1, $2...），禁止字符串拼接
- **SSL/TLS**：SSL 模式默认 require（可通过配置降级）
- **密钥管理**：数据库密码通过 configx 注入
- **审计日志**：慢查询日志（> 100ms）

## 7. 可靠性标准

- 连接池：deadpool 管理、最大连接/超时/健康检查
- 重试：resiliencx 集成（Transient 错误）
- Tx 管理：嵌套事务 savepoint
- 自动重连：连接断开自动恢复

## 8. 可维护性 — Repository trait 支持、SQL 文件化管理

## 9. 数据标准

- 时间戳：统一使用 UTC（timestamp with time zone）
- 序列化/反序列化：通过 serde + 类型映射
- 迁移管：schema 版本控制

## 10-11. 风控/回测 — N/A

## 12. 上线标准

- 灰度：先只读查询后写入
- 监控：连接数、慢查询、事务冲突率
- 告警：连接池耗尽、死锁、主从延迟

## 13. 运维标准

- 部署：SSL 证书配置、连接串管理
- 备份：DBA 层面（pg_dump/WAL）
- 恢复：PITR（时间点恢复）
- 迁移：向后兼容的 schema 变更

## 15. 交付 — features: scaffold | workspace 依赖锁定

## 16. 评分 — 4.5/5（安全标准优秀）

## 17. 验收 — 安全审查 + 性能基准 + CI门禁
