# postgresx 标准文档

**包名**：`postgresx` v0.3.13（`publish = false`）｜**角色**：PostgreSQL 池与参数化 SQL 适配器｜**状态**：foundation 可受控使用，package stable 未宣称

## 能力与边界

- 默认面为 `PostgresPool`、`PgConnection`、`PgTransaction`、`PgRepository`、`PgTxRunner`、
  `Migrator` 与 `selfcheck`（LIB-SELFCHECK §6.1）。
- deadpool wait/create/recycle、池获取、SQL 与事务终结都有内部截止时间。
- 准确事务状态为非穷尽 `TxStatus` 的 Active / Committed / RolledBack / Failed；旧三态
  `TxState` 仅为 deprecation 兼容视图。
- deprecated raw client/pool 保留一个迁移周期并分别强制脱池/返回关闭隔离池；正式路径不得依赖。
- **合同内已落地**：有界 COPY、Migrator verify/apply、远程 Require TLS（可选 CA/SNI）、
  mTLS 客户端证书、模块自检。
- **仍 OPEN / DEFER / NO-GO**：无限流式 COPY、down migration、read-replica 路由、
  查询 DSL、嵌套/跨资源事务、HA 故障切换、channel binding、服务端强制 mTLS live、
  package stable。

## 安全与可靠性

- 业务值必须使用 `$1..$N` 参数，禁止拼接不可信输入。
- 默认 `sslmode=disable` 只适合本机；远程 `disable`/`prefer` fail-closed，仅 `require` 可连接。
- `require` 使用 rustls + WebPKI roots；可叠加企业/自签 CA 与 SNI；可选客户端证书对
  （mTLS）；本版不允许 insecure 跳过校验。
- 内部/外层 timeout、future drop 或 task abort 都会永久分离未知状态连接。
- 事务进入可取消 await 前先转为 `TxStatus::Failed`，仅在连接安全恢复后回到 Active。
- 服务端语句错误使事务保持 rollback-only Failed；允许显式回滚但拒绝 COMMIT。
- Active 事务被 Drop 时关闭 session，由 PostgreSQL 回滚；不启动 fire-and-forget 任务。
- 重试仅通过显式 generic wrapper 接入；调用方必须传入 `RetrySafety`。
- `PostgresPool` 当前没有 budget 自动接线；未证明语义的 SQL 按 `UnsafeSideEffect` 处理，禁止自动多试。
- legacy async wrapper 是 unchecked compatibility，仅通过 resiliencx 统一 core 收敛 budget 错误与 attempt 观测。

## 验收

```text
cargo test -p postgresx --lib
cargo test -p postgresx --test live_postgres -- --ignored --test-threads=1
cargo test -p postgresx --test live_selfcheck -- --ignored --test-threads=1
cargo clippy -p postgresx --all-targets -- -D warnings
node scripts/postgres-deadline-conformance.mjs
cmp .agents/ssot/adapters/storage/postgres/spec/spec.md \
  .agents/ssot/adapters/storage/postgres/spec/xhyper-postgresx-complete-spec.md
```

固定摘要 PostgreSQL 17 实验覆盖池饱和、内部/外层截止、事务取消、对象脱池与新连接恢复。
live 证据覆盖 Repository / COPY / Migrator / selfcheck Full；**不**证明无限流式 COPY、
down migration、read-replica、HA 或服务端强制 mTLS。
