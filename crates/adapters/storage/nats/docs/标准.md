# natsx 标准文档

**包名**：`natsx` v0.3.2｜**角色**：NATS 存储适配器｜**状态**：声明范围内可受控使用，package stable 未宣称

## 能力与边界

- `NatsEventBus`：Core NATS at-most-once，仅实时订阅，无历史回放。
- `JetStream::publish`：等待服务端 publish ack。
- `JetStreamConsumer`：durable pull、Explicit ack、有限 fetch 与稳定投递元数据。
- `JetStreamDelivery`：显式 ack/double_ack/nak/progress/term；终结操作消费句柄。
- `term` 与 `max_deliver` 只停止重投，不会自动创建 DLQ。
- Cluster/HA、跨账户、NKey 全量、KV/ObjectStore 与自动 DLQ 均为 NO-GO。

## 编码与安全

- 使用 workspace 集中声明的 `async-nats`；旧内存实现仅在 `scaffold` feature。
- 非 loopback 默认 Require TLS；凭据与 payload 不进入 `Debug`。
- broker/协议错误保留 source；有限 fetch 的正常 expiry 与异常超时必须区分。

## 验收

```bash
cargo test -p natsx --all-targets
cargo clippy -p natsx --all-targets -- -D warnings
./scripts/broker-conformance.sh
```

单节点 conformance 只证明 Core 无回放，以及 JetStream 重投/确认/背压/终止边界；不得外推 Cluster/HA 或自动 DLQ。
