# kafkax 生产发布测试矩阵

> package `kafkax` · 驱动 **rskafka** · 版本见 `Cargo.toml`  
> 「完整测试」= 下表每行均有：**可执行入口** + **PASS/NO-GO/OOS 裁定** + **证据命令**。  
> 不把 NO-GO 能力伪装成 PASS。

## 分层

| 层 | 默认 CI | 命令 |
|----|---------|------|
| 离线功能 / 边界 | **是** | `cargo test -p kafkax --test prod_offline` |
| 单测（crate lib） | **是** | `cargo test -p kafkax --lib` |
| 集成 / 可靠性（隔离 broker） | 否（脚本） | `node scripts/kafka-prod-matrix.mjs` |
| 既有 conformance | 否 | `node scripts/kafka-broker-conformance.mjs` · `kafka-tls-sasl-conformance.mjs` |
| 受控 live | 否 | `live_event_bus` / `live_selfcheck` / `e2e_api_roundtrip` `--ignored` |
| 基准 | 否（有界） | `cargo bench -p kafkax --bench hot_path -- --quick` |
| 长稳 soak | 否（可选） | `KAFKAX_SOAK_SECONDS=60 node scripts/kafka-prod-matrix.mjs --soak` |

## 清单对照

### 0. 基准 / 功能 / 集成（基础三层）

| 类别 | 覆盖 | 入口 | 裁定 |
|------|------|------|------|
| 基准：encode_id | 吞吐/延迟摘要 | `benches/hot_path.rs` | PASS 有界 |
| 基准：produce 100B/1KiB/1MiB | 有 broker 时 | 同上 | PASS / skip 无 broker |
| 功能：配置 fail-closed | 远程明文、SCRAM、半凭据、零超时 | `prod_offline` + `config` 单测 | PASS |
| 功能：ID/offset/ALO 状态机 | 编解码、单调 checkpoint | lib 单测 | PASS |
| 集成：AMO/ALO/重复窗口 | 既有 | `broker_conformance` | PASS |
| 集成：TLS+PLAIN / 错误 CA/密码 | 既有 | `tls_sasl_conformance` | PASS |
| 集成：publish-consume / health | 既有 + prod | `live_event_bus` · `prod_reliability` | PASS |

### 1. 可靠性 / 容错

| 场景 | 入口 | 裁定 |
|------|------|------|
| Broker 不可达 | `prod_offline` connect refused；`prod_reliability` mid-flight | **PASS**（类型化错误） |
| Broker 重启窗口 | `kafka-prod-matrix.mjs --fault-restart` | **PASS**（失败可观测 + 重启后可再连） |
| 网络分区 / 自动恢复 | — | **NO-GO** 自动重连；失败结束流；宿主重建 |
| 消息不丢（publish 等确认） | produce 成功路径 + 集成 checksum | **PASS**（delivery 后返回） |
| Exactly-once / 原生幂等 | — | **NO-GO**；PTC 重复窗口有测试 |
| 分区内顺序 | `prod_reliability::partition_order` | **PASS**（单分区） |
| rebalance 期间行为 | — | **NO-GO**（无 group） |

### 2. 压力与稳定性

| 场景 | 入口 | 裁定 |
|------|------|------|
| 短时突发 concurrent produce | `prod_reliability::burst_concurrent_publish` | PASS |
| 峰值（有界 N） | bench / burst | PASS 有界 |
| 7×24 soak | `KAFKAX_SOAK_SECONDS`（默认不进 CI） | **可选**；合并门禁不强制 24h |

### 3. 背压与资源边界

| 场景 | 入口 | 裁定 |
|------|------|------|
| 有界消费队列 / close 打断 | `lifecycle` 单测 | PASS |
| 大消息 1MiB roundtrip | `prod_reliability::large_message_1mib` | PASS |
| 超大 / 非法配置 | fail-closed 配置与错误 kind | PASS |
| producer 本地 buffer 满 | rskafka 无独立 buffer API | **CLOSED**（delivery timeout → `publish_timeouts`；文档说明） |

### 4. 混沌

| 场景 | 入口 | 裁定 |
|------|------|------|
| 停 broker 容器 | `--fault-restart` | PASS 子集 |
| CPU/磁盘打满 / 时钟漂移 | — | **OOS** 本战役（可后续扩） |
| 集群 rebalance | — | **NO-GO** |

### 5. 升级与兼容

| 场景 | 入口 | 裁定 |
|------|------|------|
| 协议兼容说明 | README / 标准.md | 文档 |
| 滚动升级不丢数据 | 应用层 ALO checkpoint 恢复 | **有条件 PASS**（单 owner） |
| multi-version broker 矩阵 | — | 未宣称；单节点 harness 固定镜像 |

### 6. 安全

| 场景 | 入口 | 裁定 |
|------|------|------|
| TLS + PLAIN 正向 / 错误 CA/密码 | `tls_sasl_conformance` | PASS |
| SCRAM/OAuth/mTLS | `prod_offline` fail-closed | **NO-GO 能力 + PASS fail-closed** |
| 凭据脱敏 Debug | config 单测 | PASS |
| ACL 不足 | 错误映射 Transient/Unavailable | **CLOSED**（依赖 broker 回包；`error_map` 脱敏+kind 单测；无独立 ACL harness） |

### 7. 可观测性

| 场景 | 入口 | 裁定 |
|------|------|------|
| `stats` published/failed/closed | `prod_reliability` / pool | PASS 基础字段 |
| lag / RED 全量 | stats 基础字段 + timeouts | **PASS 有条件**（无 consumer lag；见 NO-GO group） |
| 告警触发 | 宿主侧 | OOS 库边界 |

### 8. 数据正确性 E2E

| 场景 | 入口 | 裁定 |
|------|------|------|
| 条数 + payload checksum | `prod_reliability::e2e_checksum` | PASS |
| null/空 payload | 集成 | PASS |
| schema 演进 | — | **NO-GO** schema registry |

### 9. 灾难恢复

| 场景 | 入口 | 裁定 |
|------|------|------|
| 未 ack 重建同 offset | `broker_conformance` + prod | PASS ALO |
| ack 后推进 | 同上 | PASS |
| FileOffset 落盘恢复 | conformance + offline 单测 | PASS 有条件 |
| multi-owner fencing | — | **NO-GO** |

## 诚实边界

- 默认驱动 **rskafka**，无 librdkafka group/txn。
- 单节点容器 PASS **≠** HA / Cluster stable / package stable。
- 24h soak **可选**，不是默认合并门禁。
