# kafkax 标准文档

**包名**：`kafkax` v0.3.2｜**角色**：Kafka 存储适配器｜**状态**：声明范围内可受控使用，package stable 未宣称

## 能力与边界

- `KafkaEventBus`：at-most-once；无 ack、重放或 group 保证。
- `AtLeastOnceConsumer`：手动分区、单 owner、单调应用 checkpoint；无 rebalance、lease 或 fencing。
- `ProduceThenCheckpointCoordinator`：非原子 dual-write，checkpoint 失败后重试会重复；不是 EOS。
- `FileOffsetStore`：单进程同目录 crash-consistency，不是多进程协调存储。
- 当前构建没有 Kafka TLS；请求 TLS 时必须 fail-closed。
- schema registry、consumer group、自动重连、DLQ、HA、native EOS 均为 NO-GO。

## 编码与安全

- 使用 workspace 集中声明的纯 Rust `rskafka`；旧内存实现仅在 `scaffold` feature。
- 错误映射到 `XError`，凭据与 payload 不进入日志。
- checkpoint 必须单调且拒绝负值/溢出；终端操作失败必须对调用方可见。

## 验收

```bash
cargo test -p kafkax --all-targets
cargo clippy -p kafkax --all-targets -- -D warnings
./scripts/broker-conformance.sh
```

单节点 conformance 只证明未确认重投、确认推进和非原子重复窗口；不得外推 group/rebalance/TLS/HA/EOS。
