# kafkax 标准文档

**包名**：`kafkax` v0.3.4｜**角色**：Kafka 存储适配器｜**状态**：声明范围内可受控使用，package stable 未宣称

## 能力与边界

- `KafkaEventBus`：at-most-once；无 ack、重放或 group 保证。
- `AtLeastOnceConsumer`：手动分区、单 owner、单调应用 checkpoint；无 rebalance、lease 或 fencing。
- `ProduceThenCheckpointCoordinator`：非原子 dual-write，checkpoint 失败后重试会重复；不是 EOS。
- `FileOffsetStore`：单进程同目录 crash-consistency，不是多进程协调存储。
- `SASL_SSL` 使用 rustls；支持系统/WebPKI roots、追加 PEM CA 与 SASL/PLAIN。
- 远程明文连接、错误 CA、错误密码和不完整认证配置均 fail-closed。
- connect（含 CA 装载）、控制面与 produce 均有 deadline；produce 不放大配置值。
- consumer/EventBus 队列有界；`close(deadline)` 可取消背压等待并等待在途操作释放。
- schema registry、consumer group、自动重连、DLQ、HA、native EOS 均为 NO-GO。
- draft Part2 量化栈（embedded broker / io-uring / µs 热路径）为 **OOS**。

## 编码与安全

- 使用 workspace 集中声明的纯 Rust `rskafka`；旧内存实现仅在 `scaffold` feature。
- 错误映射到稳定 `XError`；驱动原文仅用于分类，凭据、payload 与 SASL 用户名不进入公开错误或 Debug。
- checkpoint 必须单调且拒绝负值/溢出；终端操作失败必须对调用方可见。

## 验收

```text
cargo test -p kafkax --all-targets
cargo clippy -p kafkax --all-targets -- -D warnings
node scripts/kafka-broker-conformance.mjs
node scripts/kafka-tls-sasl-conformance.mjs
```

单节点 broker conformance 只证明未确认重投、确认推进和非原子重复窗口。固定摘要
TLS/SASL 实验证明受信 CA + PLAIN 可连接，错误 CA/密码 fail-closed；两者均不得外推
group/rebalance、自动重连、HA 或 EOS。
