# configx 标准文档

**包名**：`configx` v0.1.2 · **角色**：L1 进程内配置组件 · **状态**：非 Production Ready

## 1. 能力与边界

包含：

- `RwLock<HashMap<String, String>>` 内存 store；
- Memory / Env / `KEY=VALUE` File 调用时加载；
- 后源覆盖前源的分层合并；
- 调用方手动触发的原子 reload；
- 进程内 Condvar 通知；
- `secret:` 诊断脱敏。

不包含：自动 watcher、后台 runtime、远端配置中心、分布式一致性、类型化 schema、secret 加密或托管。

## 2. 原子性标准

1. 批量输入必须先在锁外完整收集。
2. `extend_pairs`、`apply_to`、`merge_into` 只允许一次写锁提交。
3. reload 必须先完成所有 source load 与 key 校验，再以一次写锁替换整个 map。
4. 失败不得清空或部分覆盖上一份有效配置。
5. 一致的多 key 读取必须使用 `try_snapshot`；多次独立 `get` 不承诺跨 generation 一致。

## 3. 错误标准

- 兼容 `get / capture` 保留 poison 折叠语义。
- 新生产路径使用 `try_get / try_snapshot / try_capture` 显式报告 poison。
- secret 与子集辅助提供 `try_get_secret / try_subset_snapshot`；兼容辅助继续折叠 poison。
- 校验与 merge 必须走 Result 快照，禁止把 poison 当作缺失或空 overlay。
- generation 使用 `checked_add`；溢出显式错误且不得修改 store 或 generation。
- 可恢复失败不得 panic。
- 用户可见 `XError` context 必须使用简体中文；关键错误测试精确断言 kind 与完整文案。
- 新公开 Result / wait API 必须提供 `# Errors` rustdoc。

## 4. 时间与生命周期标准

- `notify / reload / close` 由独立 mutation mutex 串行化。
- reload 等待 store 写锁时不得持有 state mutex；store 替换是配置线性化点。
- `wait_timeout_outcome` 使用 `try_lock` 与总 deadline，mutex 竞争不得造成无界等待。
- 显式 outcome 区分 `Changed / TimedOut / Closed`；兼容 Option API 保留。
- 接受 `Changed` 前必须二次检查 deadline；deadline 后不得推进 `seen`。
- state 可立即观察时，`Closed` 优先于同时到期的 deadline；锁竞争不得越过 deadline。
- `close` 唤醒订阅者，后续 notify 返回错误。
- crate 不自动 spawn 线程，不拥有文件 watcher 或远端轮询生命周期。

## 5. 安全标准

- `SecretString` 的 `Debug` / `Display` 脱敏。
- `ConfigSnapshot::Debug` 对 `secret:` 前缀键值脱敏。
- `MemorySource::Debug` 对 `secret:` 前缀键值脱敏。
- parse 错误不得回显原始配置行。
- 普通读取保留明文语义；脱敏不是加密或权限控制。
- 未标 `secret:` 的敏感值不保证自动脱敏。

## 6. 依赖与代码标准

- 生产依赖仅 `kernel`，无其他 L1、serde 或异步 runtime。
- 所有公开项有中文 rustdoc；标识符遵循 Rust 惯例。
- 禁止 unsafe；禁止在生产路径使用裸 `unwrap / expect`。
- 本次交付由 root 统一完成 PATCH bump 至 `0.1.2`；不得由域内执行者再次修改版本。

## 7. 测试与交付标准

必须覆盖毒锁差异、批量中点不可见、phase hook/Barrier 的 reload 锁边界、加载/校验失败保留旧值、
快照与 MemorySource secret 脱敏、parse 错误不回显、reload 锁边界与 notify 排序、state 锁竞争
deadline、deadline 后 generation、真实伪通知计数、显式 outcome、watch 溢出、关闭与正常通知。
reload 锁边界测试不得依赖轮询、sleep 或调度概率；固定树须连续运行至少 100 轮。

```text
cargo fmt -p configx -- --check
cargo test -p configx --all-targets
cargo clippy -p configx --all-targets -- -D warnings
node scripts/quality-gates/cov-gate-100.mjs -p configx --filter crates/infra/configx/src
cmp .agents/ssot/infra/configx/spec/spec.md \
    .agents/ssot/infra/configx/spec/xhyper-configx-complete-spec.md
```
