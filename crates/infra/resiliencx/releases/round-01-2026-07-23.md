## 公共 API

- 新增 `RetryContext`、`RetrySafety`、`retry_fn_safe`、`retry_async_safe`。
- 新增 `retry_async_with_budget` 与 feature `tokio` 的 `retry_async_with_deadline`。
- 新增 `retry_delay_ms_with_seed` / `apply_seeded_jitter`。

## 行为变化

- `call_with_retry_budget` 的 retry attempt 观测与其他入口对齐，预算耗尽统一返回标准 budget 错误。
- async 退避前 reserve 预算、完成后 commit；deadline 在退避期取消时 RAII refund，不泄漏预算或虚假观测。
- `BulkheadPermit` 在内部状态锁 poisoned 时仍恢复 inner 并归还容量。

## 兼容与风险

- 原有低层 sync/async retry API 保留，且继续不执行副作用安全校验。
- `RetrySafety::Idempotent` 依赖调用方保证。
- deadline 是 cooperative cancellation，不撤销已发生外部副作用。
- 本记录不构成 package stable、publish、tag 或 release 签名。
