# resiliencx 标准文档

**包名**：`resiliencx` v0.1.2 | **角色**：L1 进程内弹性原语 | **状态**：Internal Ready，非 package stable

## 1. 能力标准

- Retry：有限次数、可重试错误判定、可注入同步/异步 wait、固定/指数退避与 jitter。
- Retry Safety：`RetryContext` 聚合共同策略，生产入口要求显式 `RetrySafety`；多次尝试拒绝不安全副作用。
- Retry Budget：retry 前消费共享令牌；同步与异步耗尽统一返回标准 budget 错误。
- Generic Adapter：`call_with_retry_budget_safe` / async 版本返回 generic `T`，在首次闭包/future 前校验 safety。
- Deadline：feature `tokio` 下覆盖整次 async retry；超时映射 `DeadlineExceeded`。
- Circuit Breaker：本地三态状态机；无墙钟，Open 按拒绝次数推进 HalfOpen。
- Rate Limiter：本地令牌桶；显式 refill，资源不足立即拒绝。
- Bulkhead：本地并发计数 + RAII 许可；满载立即拒绝；毒锁恢复不得永久泄漏容量。

## 2. 安全与取消标准

- `RetrySafety::ReadOnly`：调用方保证无外部副作用。
- `RetrySafety::Idempotent`：调用方保证重复调用的领域效果等价。
- `RetrySafety::UnsafeSideEffect`：仅允许 `max_attempts == 1`。
- `RetrySafety` 是运行时策略声明，不是静态证明；调用方仍负责幂等键、事务或补偿。
- Tokio deadline 采用 cooperative cancellation；超时后不再轮询待执行 future，但不撤销已发生副作用，
  也不监管 operation 自行派生的后台任务。
- async budget 在退避前原子 reserve，退避后 commit 并记录 retry；耗尽可立即返回，退避期取消通过
  RAII refund，不会泄漏令牌或虚假观测。
- `call_with_retry_budget` / `call_with_retry_budget_async`、所有未带 `safe` 的 `retry_fn*` /
  `retry_async*` 均为 unchecked compatibility；
  生产新接线不得把它们误写成安全入口。

## 3. 观测标准

`Instrumentation::record_retry(op, attempt)` 的 `attempt` 表示“刚失败、即将触发下一次 retry 的尝试序号”，
从 1 起。仅在预算成功消费且确实准备发起下一次尝试时记录。预算耗尽不记录虚假 retry。

## 4. jitter 标准

- `apply_deterministic_jitter` / `retry_delay_ms` 为兼容入口，只依赖 attempt，结果可复现。
- attempt-only jitter 在相同配置的实例间同相，**不具备抗群聚保证**。
- `apply_seeded_jitter` / `retry_delay_ms_with_seed` 接受调用方 seed，用于实例去相关；仍非加密 RNG。
- `RetryContext::with_jitter_seed` 将 seed 接入 safe sync/async/deadline 的实际退避。

## 5. 本地立即拒绝边界

熔断、限流和舱壁不做分布式协调。限流与舱壁的 `try_*` 接口没有队列、等待 deadline、公平调度或
自动墙钟补充；失败立即返回 `Unavailable`。不得把这些原语描述为分布式限流、排队舱壁或自动冷却。

## 6. 依赖与运行时

- 普通依赖：`kernel`、`contracts`、`async-trait`；禁止直接依赖 observex。
- Tokio 仅在 feature `tokio` 下用于非阻塞 sleep 与整次 deadline。
- 库内不创建 runtime、不调用 `block_on`，不在 async 入口使用 `thread::sleep`。
