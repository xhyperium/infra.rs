# kernel 标准文档

**包名**：`kernel` v0.3.0 | **角色**：L0 语义信任根 | **层级**：L1+L4

---

## 1. 需求标准

### 目标

kernel 定义全系统必须唯一且长期稳定的三类语义：错误分类与响应、时间获取与表示、生命周期与关停信号。kernel 不提供配置、日志、网络、异步运行时、依赖注入、持久化或业务能力。

### 边界

- **包含**：`error`（XError/ErrorKind/XResult）、`clock`（Clock/Timestamp/MonotonicInstant/SystemClock）、`lifecycle`（ComponentState/ShutdownSignal/ShutdownGuard）
- **不包含**：配置、日志、网络、异步运行时、依赖注入、持久化、业务逻辑
- **依赖**：仅 `thiserror`（生产依赖），`loom`（cfg(loom) 测试）

### 输入输出

| 模块 | 核心类型 | 输入 | 输出 |
|------|----------|------|------|
| error | XError | 错误分类（ErrorKind）+ 上下文信息 | 不透明错误 + kind/context 查询方法 |
| clock | Clock trait | 时间获取请求 | Timestamp/MonotonicInstant |
| clock | SystemClock | 系调用 | 墙钟 Unix 纳秒 / 单调时钟纳秒 |
| lifecycle | ShutdownSignal | trigger() 调用 | 信号触发，多方 wait() 可观测 |
| lifecycle | ComponentState | try_transition() 状态转换 | 合法转换/非法转换错误 |

### 约束

- `#![forbid(unsafe_code)]` — 零 unsafe
- `#![deny(missing_docs)]` — 所有公开项必须文档
- `#![deny(unreachable_pub)]` — 禁止未预期的公开可见性
- 任何新增公开项、依赖或 feature 都必须走 RFC
- 公开 API 棘轮门禁（`check-public-api.mjs`）
- loom CI 并发模型验证
- 100% 行覆盖率门禁
- mutants/miri CI

## 2. 设计标准

### 架构

kernel 采用平层三模块架构：

```text
kernel
├── error   — 错误分类系统（不透明 XError + 9 种 ErrorKind）
├── clock   — 时间抽象（Clock trait + SystemClock + Timestamp + MonotonicInstant）
└── lifecycle — 生命周期管理（ComponentState + ShutdownSignal）
```

根 crate re-export 所有公开符号，亦可 `kernel::error`/`kernel::clock`/`kernel::lifecycle` 直接使用。

### 模块划分

| 模块 | 职责 | 公开 trait | 公开类型 |
|------|------|------------|----------|
| error | 错误分类与响应 | — | XError, ErrorKind, XResult, BoxError |
| clock | 时间获取与表示 | Clock | SystemClock, Timestamp, MonotonicInstant, ClockDomain, ClockError |
| lifecycle | 生命周期与关停 | — | ComponentState, ShutdownSignal, ShutdownGuard, LifecycleError |

### 接口

- trait 设计最小化：Clock 仅定义 `now()` 和 `monotonic()` 两个方法
- XError 采用不透明模式（opaque error）— 外部只通过 kind/context 查询方法访问，不暴露内部变体
- ShutdownSignal 采用 `new → (ShutdownGuard, ShutdownSignal)` 拆分模式 — guard 仅触发一次、signal 多方观察
- From `ClockError` for XError 全部映射为 `ErrorKind::Unavailable`，保留 source

### 数据流

```text
外部代码 → kernel::Clock.now() → SystemClock::now() → Timestamp(unix_nanos)
外部代码 → XError::invalid(msg) → XError { kind, context, source? }
外部代码 → ShutdownSignal::new() → (guard, signal) → guard.trigger() → signal.is_triggered()
```

## 3. 编码标准

### 命名

- 类型：UpperCamelCase — `XError`, `ErrorKind`, `SystemClock`, `Timestamp`, `ShutdownSignal`
- 函数/方法：snake_case — `now()`, `monotonic()`, `is_retryable()`, `trigger()`
- 模块：snake_case — `error`, `clock`, `lifecycle`
- 常量/变体：UpperCamelCase — `ErrorKind::Invalid`, `ComponentState::Running`
- ErrorKind 变体名采用形容词/过去分词形式：`Invalid`, `Missing`, `Conflict`, `Transient`, `Unavailable`, `Cancelled`, `DeadlineExceeded`, `Invariant`, `Internal`

### 风格

- 严格遵循 `cargo fmt` 默认风格
- `#[must_use]` 标注所有返回 XResult 或重要结果的函数
- `#![forbid(unsafe_code)]` — 不允许 unsafe
- `#![deny(missing_docs)]` — 所有公开项必须提供 rustdoc
- 所有公开类型实现 `Debug + Clone + Send + Sync` 等必要 trait

### 错误处理

- 使用 XError 作为 crate 唯一公开错误类型
- 内部 ClockError/ShutdownError 通过 From 实现映射
- 所有 fallible 函数返回 `XResult<T>`（即 `Result<T, XError>`）
- ErrorKind 九种分类覆盖所有场景：Invalid(参数非法)、Missing(不存在)、Conflict(冲突)、Transient(可重试)、Unavailable(不可用)、Cancelled(取消)、DeadlineExceeded(超时)、Invariant(不变量违例)、Internal(内部错误)

### 日志

- kernel 作为 L0 信任根，不直接依赖 tracing 等日志框架
- 日志由上层 crate 在调用 kernel 函数时添加

### 注释

- 所有公开项必须提供完整 rustdoc（说明、Panics、Errors、Examples）
- 使用中文/英文均可，但必须清晰完整
- 内部实现仅复杂逻辑需要注释
- rustdoc 示例应可直接作为 cargo test 运行

## 5. 性能标准

### 延迟

| 操作 | 目标 | 说明 |
|------|------|------|
| XError::invalid | < 100ns | 单次构造开销 |
| Clock::now | < 200ns | 系统调用开销 |
| ShutdownSignal::trigger | < 100ns | 原子标志位写入 |
| ShutdownSignal::is_triggered | < 50ns | 原子标志位读取 |

### 吞吐

所有操作均为单次调用级性能，不涉及批量处理。

### 资源占用

- kernel 本身无内存分配（XError 上下文分配除外）
- 每个 XError 实例通常 < 100 bytes 堆分配
- 零启动开销

### 发能力

- XError: Send + Sync，无内部可变状态
- ShutdownSignal: 内部 AtomicBool，无锁并发
- SystemClock: 内部无状态

## 7. 可靠性标准

### 容错

- 所有错误通过 XError 分类，调用方可按 ErrorKind 决策响应
- ErrorKind::Transient/Unavailable 标识可重试错误
- 没有 panic 路径（公开 API 均返回 Result）

### 重试

- kernel 本身不实现重试逻辑（L0 信任根）
- 通过 ErrorKind::Transient 标识告知调用方需重试
- `XError::retry_after()` 提供可选的推荐等待时间

### 降级

- kernel 不涉及降级（L0 信任根无替代实现）

### 幂等

- clock.now() 调用天然幂等（读操作）
- ShutdownSignal.trigger() 幂等 — 二次触发无副作用

### 异常恢复

- SystemClock::now() 失败返回 ClockError::Unavailable，而非 panic
- ShutdownSignal 信号触发不可逆（by design）

## 9. 数据标准

### 数据源

- Timestamp 来自系统时钟（墙钟）/ 平台单调计数器（单调钟）
- 时钟数据由 OS 内核提供

### 数据清洗

- Timestamp: from_unix_nanos 拒绝 Unix 纪元前时间 → ClockError::BeforeUnixEpoch
- MonotonicInstant: 仅通过 Clock 接口创建，不提供公共构造器

### 数据对齐

- 所有时间戳统一使用 Unix 纳秒作为内部表示
- 系统墙钟与单调钟分离，各自独立

### 缺失值处理

- 时钟不可用时返回 ClockError::Unavailable
- 不提供默认值回退

### 时区

- Timestamp 始终为 UTC（Unix 纳秒）
- 不含时区偏移信息
- 时区转换由上层调用方处理

### 精度

- Timestamp: 纳秒精度（i128 内部表示）
- MonotonicInstant: 纳秒精度
- 系统时钟可能低于纳秒精度（取决于硬件和 OS）
