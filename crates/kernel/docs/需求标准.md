# 需求标准

**适用于**：`crates/kernel` | **包名**：`kernel` | **层级**：L0（语义信任根）

---

## 1. 需求标准

### 目标

`kernel` 定义全系统必须唯一且长期稳定的三类语义，作为 infra.rs 的**语义信任根**：

1. **错误分类与响应**（`error` 模块）——按"调用方应如何反应"分类错误，而非按模块来源分类。调用方不得通过字符串匹配决定是否重试、降级或 fail-fast。
2. **时间获取与表示**（`clock` 模块）——墙钟与单调钟分离，时间源必须显式注入。墙钟允许回退，单调钟不透明不可持久化，跨 domain 比较返回 `None`。
3. **生命周期与关停信号**（`lifecycle` 模块）——关停一次触发、多方观察、不可逆。

### 边界

- **包含**：
  - 9 种 `ErrorKind`Invalid/Missing/Conflict/Transient/Unavailable/Cancelled/DeadlineExceeded/Invariant/Internal）+ `XError` 不透明封装
  - `Timestamp`（i64 纳秒 Unix 时间戳）+ `MonotonicInstant`（不透明间隔采样点）+ `ClockDomain`
  - `Clock` trait（`now` + `monotonic` 双通道）+ `SystemClock` 生产实现
  - `ShutdownSignal` / `ShutdownGuard` 关停协议 + `ComponentState` 6 状态机
  - `ClockError` / `LifecycleError` 中文 Display

- **不包含**：
  - 配置、日志、网络（tokio/async）、异步运行时、依赖注入、持久化、业务能力
  - serde Serialize/Deserialize 实现（wire 由协议层版本化）
  - `Component` trait（已废弃，通过 rustdoc compile_fail 禁止）
  - 通用 `prelude` 模块或第三方 re-export
  - 分布式一致性、跨进程时钟同步

- **依赖**生产依赖仅 `thiserror`；dev 依赖 `proptest` / `static_assertions` / `serde`（仅负向面）；loom 通过 `cfg(loom)` 条件编译。

### 输入输出

| 符号 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `XError::invalid` | 上下文描述 | `XError` (Invalid) | 参数/输入非法 |
| `XError::transient_after` | 上下文 + `Duration` | `XError` (Transient) | 瞬时故障含重试提示 |
| `Clock::now` | `&self` | `Result<Timestamp, ClockError>` | 墙钟时间 |
| `Clock::monotonic` | `&self` | `MonotonicInstant` | 单调采样，不可失败 |
| `ShutdownSignal::wait_timeout` | `Duration` | `bool`（是否已触发） | 带超时等待关停 |
| `Timestamp::checked_add` | `Duration` | `Option<Timestamp>` | 安全加法 |
| `ComponentState::try_transition` | `ComponentState` | `Result<(), LifecycleError>` | 状态转换 |

### 约束

- **零 unsafe 代码**：`#![forbid(unsafe_code)]`
- **零生产依赖**（除 `thiserror`）：无 tokio/serde/chrono/anyhow 等
- **零 feature flags**：`default = []`
- **字段私有**：`XError`、`Timestamp` 内部字段不可被下游直接访问
- **无 `Default`**：`Timestamp`/`MonotonicInstant`/`ErrorKind` 不实现 `Default`
- **无 `From<&str>`/`From<String>`**：禁止通过字符串旁路错误语义
- **无 `not_found`/`other`**：禁止模糊错误类别
- **负向面测试**：rustdoc `compile_fail` 和 `static_assertions` 编译期合同验证
- **CI 门禁**：line ≥95%、branch ≥90%、mutants ≥90%、miri 周调度、loom PR/push 门禁
