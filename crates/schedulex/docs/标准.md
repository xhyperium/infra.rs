## 1. 需求标准

### 目标

schedulex 提供进程内任务调度能力：任务 ID 登记表（in-memory task ID registry）+ 滴答驱动的 JobRunner（tick-driven, minimal cron）。

### 边界

- **包含**：TaskId 分配与注册、JobRunner 调度执行
- **不包含**：分布式调度、持久化任务队列、跨进程任务编排
- **依赖**：仅 std（无生产依赖）

### 核心接口

| 符号 | 说明 |
|------|------|
| TaskId | 进程内任务唯一标识 |
| JobRunner | tick-driven 任务执行器 |

### 约束

- std-only，零生产依赖
- 非分布式调度器
- 滴答驱动而非 wall clock 驱动

### 设计标准

- 简单两层架构：TaskId 注册层 + JobRunner 执行层

### 编码标准

- std-only 风格，无异步运行时依赖
- Tick 驱动模式需正确的时间计算

### 测试标准

- TaskId 唯性测试
- JobRunner 滴答调度测试（精确时间点）
- 边界：零任务、大任务量、任务取消

|------|----------|
| TaskId::new | < 100ns |
