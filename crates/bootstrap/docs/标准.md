## 1. 需求标���

### 目标

bootstrap 提供进程启动时的依赖组装（R3.1 exemption / ADR-016），是类型安全的组合根（composition root）。

### 边界

- **包含**：StoreSet（存储集合）、AsyncDrain（异步清理）、Bounded 类型安全注入（venue/storage）
- **不包含**：String/Any/TypeId Service Locator、运行时服务发现、IoC 容器
- **依赖**：kernel、contracts、observex、evidence

### 约束

- ADR-016: 唯一组合根
- 禁止 Service Locator 模式
- 所有注入为类型安全（Bounded* 泛型）

## 2. 设计标准

### 架构

```text
bootstrap
├── stores      — StoreSet（统一管理所有存储后端）
├── drain       — AsyncDrain（优雅关闭）
├── venue       — BoundedVenue（交易所注入）
└── storage     — BoundedStorage（存储注入）
```text
