# observex 标准文档

**包名**：`observex` v0.1.1 | **角色**：L1 可观测性基础设施 | **层级**：L1 Internal Ready

---

## 1. 需求标准

### 目标

observex 提供 L1 可观测性基础设施，包括 tracing/metrics 框架集成 + OTEL-compatible 进程内导出能力。

### 边界

- **包含**：TracingInstrumentation（基于 tracing crate 的 span 追踪）、InMemoryExporter（内存导出器，用于测试）、ExportingInstrumentation（flush/shutdown 集成）
- **不包含**：完整 OpenTelemetry SDK 实现、OTLP 远程导出到 collector、metrics 聚合器
- **依赖**：kernel、contracts、tracing（生产依赖）

---

## 2. 设计标准

### 架构

```text
observex
├── instrumentation   — TracingInstrumentation（span 创建/结束 + 属性注入）
├── exporter          — InMemoryExporter（内存存储已导出 span）
├── export_instr      — ExportingInstrumentation（flush/shutdown 生命周期管理）
└── types             — span 类型定义
```

### 数据流

```text
应用代码 → TracingInstrumentation.span("operation") → tracing::span!
         → Instrumentation.instrument(future) → span 包裹 future
         → InMemoryExporter.export(span) → 内存存储
         → ExportingInstrumentation.flush() → 导出所有待处理 span
```
