## 1. 需求标准

### 目标

testkit 提供跨 crate 共享的确定性测试基础设施，主要包括 `ManualClock`（手动时钟，用于控制时间流）和 `IntegrationHarness`（集成测试编排）。

### 边界

- **包含**：ManualClock（ControlClock trait + 手动时间推进）、IntegrationHarness（测试编排辅助）
- **不包含**���生产运行时功能、特定 crate 的测试用具
- **依赖**：kernel（生产依赖）；proptest、static_assertions（开发依赖）

### 输入输出

| 符号 | 输入 | 输出 | 说明 |
|------|------|------|------|
| ManualClock::now | — | Timestamp | 返回当前手设时间 |
| ManualClock::set | Timestamp | — | 直接设置当前时间 |
| ManualClock::advance | Duration | — | 推进指定时间量 |
| ManualClock::monotonic | — | MonotonicInstant | 返回当前单调时间 |

### 约束

- T0 dev-dep 仅用于测试，不引入生产代码
- SPEC-TESTKIT-002 明确定义契约

## 2. 设计标���

### 架构

```text
testkit
└── ManualClock implements Clock trait
```

ManualClock 实现 kernel::clock::Clock trait，提供可控制的时间源。

### 模块划分

| 模块 | 职责 | 公开类型 |
|------|------|----------|
| ManualClock | 手动控制的时间实现 | ManualClock implements Clock |

### 接口

- 实现 kernel 的 Clock trait（now() + monotonic()）
- 扩展方法：set() / advance()

### 数据流

```text
测试代码 → ManualClock.set(ts) → clock.now() → ts
测试代码 → ManualClock.advance(dur) → clock.now() → ts+dur
```

## 3-9（与 kernel 类似的框架性标准从简）

> testkit 作为 T0 测试支持 crate，各项标准从简，仅覆盖关键差异点。

### 编码标准

- 遵循 kernel 的编码约定
- 公开项必须提供 rustdoc
- ManualClock 内部使用 AtomicU64/Mutex 管理可变时间状态

### 测试标准

- 单元测试覆盖所有 ManualClock 公开方法
- 边界：时间回退、零推进、大跨度推进、溢出边界
- IntegrationHarness 的端到端集成测试
- proptest 属性测试覆盖时间推进路径

### 安全标准

- N/A（纯测试工具，无生产安全风险）

### 可靠性标准

- ManualClock: Send + Sync，内部 Mutex 保护
- 不需要重试/降级（测试基础设施）
