# decimalx 标准文档

**包名**：`decimalx` v0.1.2 | **角色**：/types/ 十进制数值类型 | **层级**：L1 checked path Internal Ready

---

## 1. 需求标准

### 目标

decimalx 提供金融级高精度十进制算术类型，是系统唯一定义 Decimal 的地方（ADR-007）。禁止 f32/f64 参与金额/数量运算。

### 边界

- **包含**：Decimal 类型、checked 算术、serde 序列化、舍入策略、scale 管理
- **不包含**：业务逻辑、汇率/外汇换算、聚合定价或跨语言精确 wire 协议
- **依赖**：kernel、serde

### 核心接口

| 符号 | 说明 |
|------|------|
| Decimal | 核心十进制类型（field-private） |
| Decimal::try_new(i128, scale) | 构造（校验 scale ≤ MAX_SCALE） |
| Decimal::checked_add/sub/mul/div | checked 算术 |
| RoundingStrategy | 舍入策略枚举 |
| DecimalError | scale/mantissa/除零/舍入/表示范围错误 |

### 约束

- `#![forbid(unsafe_code)]`
- `#![deny(missing_docs)]`
- 字段私有，非法 scale 不可表示
- serde 反序列化走校验路径
- 资金路径必须用 checked API
- feature `panicking-ops` 默认关闭

## 2. 设计标准

### 架构

```text
decimalx
└── Decimal (i128 mantissa + u8 scale) + RoundingStrategy + DecimalError
```

### 关键设计决策

- i128 + u8 内部表示，scale ≤ MAX_SCALE
- 双目运算：scale 不一致时对齐到较大 scale（补零不隐式舍入）
- 除法必须显式指定 RoundingStrategy
- 中间值 i128 溢出即 Err（即使约分后可表示）

### 数据流

```text
parse("123.45") → Decimal{mantissa: 12345, scale: 2}
try_new(12345, 2) → Ok(Decimal)
checked_add(a, b) → Result<Decimal, DecimalError>
serde → {"mantissa": 12345, "scale": 2} (WIRE_SCHEMA_VERSION 标识)
```
